# 2. Definitions

> 定义

For the purposes of this standard, the following terms and definitions apply.

> 为了本标准的目的，下列术语和定义适用。

The TOGAF® framework [[4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Referenced-docs.html#Ref4)] should be referenced for Enterprise Architecture-related terms not defined in this chapter.

> TOGAF®框架[[4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Referenced-docs.html#Ref4)]可以参考本章中没有定义的企业架构相关术语。

Merriam-Webster’s Collegiate Dictionary (11th Edition) should be referenced for all other terms not defined in this chapter.

> 本章中未定义的所有其他术语应参考韦氏大学词典(第11版)。

---

Any conflict-冲突,争执 between definitions described here and the TOGAF framework is unintentional-非故意的,无意识的.

> 这里描述的定义与TOGAF框架之间的任何冲突都是无意的。

If the definition of a term is specific to the ArchiMate modeling language, and a general definition is defined by the TOGAF framework, then this is noted in the definition.

> 如果术语的定义是特定于ArchiMate建模语言的，而通用定义是由TOGAF框架定义的，则在定义中注明这一点。



## 2.1. ArchiMate Core Framework

> 建立核心框架

A reference structure used to classify-把…分类 elements of the ArchiMate core language.

> 一种参考结构，用于对原始核心语言的元素进行分类。

It consists of three layers and three aspects.

> 它包括三个层次和三个方面。

**NOTE：**The ArchiMate Core Framework is defined in detail in [Section 3.4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework).

> **注:** ArchiMate 核心框架的详细定义见 [Section 3.4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework)。



## 2.2. ArchiMate Core Language

> 核心语言

The central part of the ArchiMate language that defines the concepts to model Enterprise Architectures.

> ArchiMate 语言的中心部分，它定义了对企业架构建模的概念。

It includes concepts from three layers: Business, Application, and Technology (including Physical).

> 它包括来自三个层的概念:业务、应用程序和技术(包括物理)。



## 2.3. Architecture View

> 体系结构视图

A representation of a system from the perspective of a related set of concerns-关注点.

> 从一组相关关注点的角度对系统的表示。

**NOTE：**In some sections of this standard, the term “view” is used as a synonym-同义词 for “architecture view”.

> **注: ** 在本标准的某些章节中，术语“视图”被用作“架构视图”的同义词。



## 2.4. Architecture Viewpoint

> 体系结构的观点

A specification-规格,规范 of the conventions-约定,协定 for a particular kind of architecture view.

> 特定类型架构视图的约定规范。

**NOTE：**In some sections of this standard, the term “viewpoint” is used as a synonym-同义词 for “architecture viewpoint”.

> **注:**在本标准的某些章节中，术语“视点”被用作“架构视点”的同义词。



## 2.5. Aspect

> 方面

Classification-分类,分级 of elements based on layer-independent-独立的 characteristics related to the concerns of different stakeholders.

> 基于与不同利益相关者的关注点相关的层无关特征的元素分类。

Used for positioning-定位 elements in the ArchiMate meta model.

> 用于定位 ArchiMate 元模型中的元素。

See also [Section 1.3](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Introduction.html#sec-Conformance).

**NOTE：**Aspects are described in [Section 3.4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework).

> **注: ** 方面在 [3.4节](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework) 中进行了描述。



## 2.6. Attribute

> 属性

A property associated with an ArchiMate language element or relationship.

> 与 ArchiMate 元素或关系相关联的属性。



## 2.7. Composite-复合的 Element

> 复合单元

An element consisting of other elements from multiple aspects or layers of the language.

> 由来自语言的多个方面或层次的其他元素组成的元素。



## 2.8. Concept

> 概念

Either an element, a relationship, or a relationship connector.

> 元素、关系或关系连接器。

See also [Section 2.12](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Definitions.html#sec-Element) and [Section 2.14](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Definitions.html#sec-Model).

**NOTE：**The top-level language structure is defined in detail in [Section 3.2](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-Top-Level-Language-Structure).

> **注:**顶层语言结构在 [Section 3.2](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-Top-Level-Language-Structure) 中有详细的定义。



## 2.9. Conformance

> 一致性

Fulfillment of specified-指定 requirements.

> 完成指定的要求。



## 2.10. Conforming Implementation

> 符合标准的实现

An implementation which satisfies the conformance requirements defined by the conformance clause of this standard.

> 满足本标准一致性条款所定义的一致性要求的实现。

See [Section 1.3](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Introduction.html#sec-Conformance).



## 2.11. Core Element

> 核心元素

A structure-结构 or behavior element in one of the core layers of the ArchiMate language.

> ArchiMate 语言核心层中的一种结构或行为元素。

**NOTE：**Core elements are described in detail in [Section 3.4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework).

> **注:** 核心元素在 [Section 3.4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-The-ArchiMate-Core-Framework) 中有详细描述。



## 2.12. Element

> 元素

Basic unit in the ArchiMate metamodel.

> ArchiMate 元模型中的基本单元。

Used to define and describe the constituent-组成的,构成的 parts of Enterprise Architectures and their unique-独特的,唯一 set of characteristics.

> 用于定义和描述企业架构的组成部分及其独特的特征集。



## 2.13. Layer

> 层，层次

An abstraction of the ArchiMate framework at which an enterprise can be modeled.

> 可以对企业进行建模的 ArchiMate 框架的抽象。



## 2.14. Model

> 模型

A collection of concepts in the context of the ArchiMate language structure.

> 在 ArchiMate 语言结构上下文中的概念集合。

**NOTE：**The top-level language structure is defined in detail in [Section 3.2](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-Top-Level-Language-Structure).

> **注:** 顶层语言结构在 [Section 3.2](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Language-Structure.html#sec-Top-Level-Language-Structure) 中有详细的定义。

For a general definition of model, see the TOGAF framework [[4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Referenced-docs.html#Ref4)].

> 对于模型的一般定义，请参见TOGAF框架[[4](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Referenced-docs.html#Ref4)]。



## 2.15. Relationship

> 关系

A connection between a source and target concept.

> 源概念和目标概念之间的连接。

Classified-把…分类,把…分级 as structural, dependency, dynamic, or other.

> 分类为结构的、依赖的、动态的或其他的。

**NOTE：**Relationships are defined in detail in [Chapter 5](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Relationships-and-Relationship-Connectors.html).

> **注: ** 关系的详细定义在 [Chapter 5](https://pubs.opengroup.org/architecture/archimate3-doc/ch-Relationships-and-Relationship-Connectors.html)。



## 2.16. Relationship Connector

> 关系连接器

A concept that connects two or more relationships of the same type.

> 连接两个或多个相同类型关系的概念。
